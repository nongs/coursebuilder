## 코스빌더 요구사항 정리

### 1. 개요
- **목표**: 강의 저작도구(코스빌더)를 웹 애플리케이션 형태로 구현한다.
- **데이터 저장 방식**: 별도 DB는 사용하지 않고, **`localStorage`를 이용해 DB와 통신하는 것처럼 표현**한다.
  - 실제 DB 연동을 흉내 내기 위한 비동기 API 래퍼를 만들고, 내부 구현만 `localStorage`를 사용한다.
  - 일반적인 캐시 개념으로도 `localStorage`를 활용한다.

### 2. 기술 스택
- **프론트엔드 프레임워크**: React
- **번들러/개발 서버**: Vite (React + TypeScript 템플릿 기반)
- **언어**: TypeScript
- **상태 관리**: Zustand
- **드래그 앤 드롭**: `@dnd-kit/core`, `@dnd-kit/sortable`
  - 여러 개의 독립된 드롭존(영역)을 만들 수 있어야 한다.
  - 서로 다른 영역끼리는 요소가 침범/이동하지 못하도록 제어한다.
  - 같은 계층 레벨 내에서는 순서 변경(재정렬)이 가능해야 한다.

### 3. 도메인 모델 구조
- **전체 계층 구조**
  - `강의 (Course)` > `주차 (Week)` > `챕터 (Chapter)` > `페이지 (Page)` > `학습요소 (LearningItem)`
- **ID 기반 참조 방식**
  - 각 엔티티는 고유 `id`를 가진다.
  - 상위 엔티티는 하위 엔티티의 **id 리스트를 소유**한다.
    - 예: `Course` 안에 `weekIds: string[]`
    - `Week` 안에 `chapterIds: string[]`
    - `Chapter` 안에 `pageIds: string[]`
    - `Page` 안에 `learningItemIds: string[]`
  - 실제 상세 데이터는 별도의 `byId` 맵 형태로 관리 (`Record<string, Entity>`).

### 4. 드래그 앤 드롭 요구사항
- **여러 레벨에서 드래그 앤 드롭 사용**
  - 주차 정렬
  - 챕터 정렬
  - 페이지 정렬
  - 페이지 내 학습요소 정렬
- **영역 간 경계**
  - 각 레벨(주차/챕터/페이지/학습요소)은 **서로 다른 드롭존**으로 본다.
  - 같은 레벨끼리만 재정렬/이동 가능하게 제한한다.
- **목록 안의 목록 구조**
  - 예: 주차 리스트 안에 주차 카드, 그 안에 챕터 리스트, 그 안에 페이지 리스트…
  - 중첩 구조가 깊어지더라도, 같은 레벨 기준으로만 순서 변경이 가능해야 한다.

### 5. 상태 관리 및 저장 전략
- **Zustand 스토어**
  - 코스 전체 트리를 메모리 상에서 관리한다.
  - 예시:
    - `coursesById`, `weeksById`, `chaptersById`, `pagesById`, `learningItemsById`
    - `rootCourseId` 또는 다수 코스를 지원하는 경우 `courseIds: string[]`
  - CRUD 및 드래그 앤 드롭에 따른 순서 변경 액션을 제공한다.
- **영속화(API 형태 + localStorage)**
  - `@api/courseApi`: `fetchCourseTree()`(진입·새로고침), `saveCourseTree()`(저장 버튼).
  - 내부 구현은 `localStorage`에 JSON 직렬화/파싱만 수행(실제 DB 대체).
  - 편집은 메모리(Zustand)에만 반영되고, **저장 버튼**으로만 영속화된다.
  - 로드/저장 중 전역 로더(`GlobalLoader`)로 화면 차단.

### 6. UI/UX 방향
- **레이아웃**
  - 좌측: 코스 구조 트리(주차/챕터/페이지) 네비게이션
  - 중앙: 선택된 페이지 및 학습요소 편집 영역
  - 우측(선택): 속성 패널 또는 미리보기
- **드래그 앤 드롭 UX**
  - 드래그 중인 카드에 시각적 강조(그림자, 배경색).
  - 드롭 가능한 영역에 하이라이트 표시.
  - 잘못된 영역으로 드롭 시 원위치로 스냅되도록 처리.

### 7. 향후 확장 아이디어 (선택 사항)
- 여러 코스 관리(코스 목록 화면 추가).
- 버전 관리(스냅샷 저장, 되돌리기).
- 실서비스 DB 연동 시를 고려한 API 인터페이스 분리.
